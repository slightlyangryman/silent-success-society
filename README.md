# Silent Success Society — Initiation Guide

> 신규 참여자 자기진단 및 실행 가이드 웹앱

남의 기준이 아니라 나의 기준을 찾고, 거창한 선언이 아니라 오늘부터 반복할 수 있는
작은 행동을 설계하기 위한 비공개 자기진단 웹앱입니다.

---

## 1. 프로젝트 소개

이 앱은 단순한 설문지가 아닙니다.

- 78개 문항으로 현재 자신의 **행동 패턴**을 진단하고,
- 작심삼일을 반복하는 이유를 **시스템 관점**에서 이해하고,
- 결과로부터 **오늘 당장 실행할 수 있는 작은 행동**까지 설계하도록 돕는
  온보딩 흐름입니다.

콘텐츠는 두 축에 기반합니다.

- *Die Empty* — Mapping(설계) / Making(실행) / Meshing(성장)의 3축과 7가지 장애물
- *아주 작은 습관의 힘* — 시스템, 정체성, 2분 법칙, 신호–행동–보상

이 앱은 종교, 정치, 사상, 투자 권유, 영업, 포교와 무관합니다.
사용자를 비난하거나 낙인찍지 않고, "의지가 부족하다"가 아니라
"아직 시스템이 작게 설계되지 않았다"로 해석합니다.

## 2. 핵심 기능

- **개인정보 없는 익명 진단** — 이름·전화번호·이메일·소속 등 일체 수집하지 않음
- **간단한 비밀번호 게이트** — 링크를 받은 사람만 입장
- **78문항 12영역 자기진단** — 3M, 7장애물, 습관 시스템, 조직 활동 준비도
- **5가지 행동 유형 판정** — Developer / Driver / Drifter / Dreamer / Starter
- **작심삼일 위험도 계산** — 낮음 / 보통 / 높음 / 매우 높음
- **장애물별 처방** — 현재 상태, 자주 드는 생각, 작동 방식, 처방, 2분 행동
- **실행계획 작성** — 정체성 문장, 2분 행동, 신호, 보상, Step/Sprint/Stretch
- **결과 요약 복사 / 인쇄(PDF 저장)**
- **localStorage 자동 저장** — 새로고침해도 답변·계획 유지
- **모바일 우선 반응형 UI**

## 3. 기술 스택

- **Next.js 14** (App Router)
- **TypeScript 5**
- **Tailwind CSS 3**
- **React 18 hooks** 기반 상태 관리 (별도 전역 라이브러리 없음)
- **localStorage / sessionStorage**
- **Vercel** 배포
- (선택) **Supabase** — 익명 저장 옵션, 환경변수 없으면 비활성화

블랙홀 hero 이미지는 외부 이미지 파일 없이 순수 CSS(`radial-gradient`)로 구현되어
있어 별도 자산을 준비하지 않아도 작동합니다.

## 6. 비밀번호 설정 방법

이 앱은 회원가입을 만들지 않고, 환경변수 기반 비밀번호 게이트만 제공합니다.

루트에 `.env.local` 파일을 만들고 다음 변수를 설정합니다.

```bash
NEXT_PUBLIC_APP_PASSWORD=silent2026
```

- 비밀번호를 변경하려면 위 값을 바꾸고 서버를 다시 시작하세요.
- 인증 상태는 `sessionStorage`에 저장됩니다. 브라우저를 완전히 닫으면 다시 묻습니다.

### ⚠️ 보안 주의사항

**이 비밀번호 기능은 강력한 보안이 아니라 단순한 접근 제한용입니다.**

- 환경변수가 `NEXT_PUBLIC_` 접두사를 사용하므로 클라이언트 번들에 포함됩니다.
- 즉, 개발자 도구로 확인하면 비밀번호 값을 볼 수 있습니다.
- "신뢰할 수 있는 사람들에게만 링크를 공유하는" 수준의 가벼운 차단용으로만
  적합합니다.
- 진짜 보안이 필요하다면 Next.js 미들웨어, Edge Functions, 또는 정식 인증
  (NextAuth, Clerk 등)을 추가로 도입해야 합니다.

또한, **비밀번호 입력 기록은 어떤 형태로도 저장하지 않습니다.**

## 7. Vercel 배포 방법

1. 코드를 GitHub(또는 GitLab/Bitbucket) 저장소에 푸시합니다.
2. [Vercel 대시보드](https://vercel.com/new)에서 **New Project**를 선택하고
   해당 저장소를 임포트합니다.
3. **Environment Variables** 섹션에서 다음 값을 추가합니다.
   - `NEXT_PUBLIC_APP_PASSWORD` — 원하는 비밀번호
   - (선택) `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. **Deploy** 버튼을 누릅니다.
5. 배포가 완료되면 발급된 URL을 신뢰할 수 있는 신규 참여자에게만 비밀번호와
   함께 공유합니다.

robots 메타가 `noindex, nofollow`로 설정되어 있어 기본적으로 검색엔진에
노출되지 않습니다.

## 8. 설문 구조

총 **78문항, 12영역**으로 구성됩니다.

| 영역 | 문항 수 | 해석 방향 |
|---|---|---|
| Mapping / 설계 | 7 | 점수가 높을수록 좋음 |
| Making / 실행 | 7 | 점수가 높을수록 좋음 |
| Meshing / 성장 | 7 | 점수가 높을수록 좋음 |
| Aimlessness / 목적 상실 | 6 | 점수가 높을수록 장애물이 강함 |
| Boredom / 지루함 | 6 | 점수가 높을수록 장애물이 강함 |
| Comfort / 안락함 | 6 | 점수가 높을수록 장애물이 강함 |
| Delusion / 자기기만 | 6 | 점수가 높을수록 장애물이 강함 |
| Ego / 자존심 | 6 | 점수가 높을수록 장애물이 강함 |
| Fear / 두려움 | 6 | 점수가 높을수록 장애물이 강함 |
| Guardedness / 닫힌 태도 | 6 | 점수가 높을수록 장애물이 강함 |
| 습관 시스템 | 7 | 점수가 높을수록 좋음 |
| 조직 활동 준비도 | 7 | 점수가 높을수록 좋음 |

각 문항은 **1점(전혀 아니다) ~ 5점(매우 그렇다)** 의 5점 척도입니다.
역문항(`reverse: true`)은 `6 - 입력값`으로 환산됩니다.

문항 데이터는 `data/questions.ts`에 분리되어 있어 쉽게 수정/확장할 수 있습니다.

## 9. 점수 계산 방식

핵심 로직은 모두 `lib/scoring.ts`에 있습니다.

### 영역 점수
각 영역 문항의 평균(역문항은 환산 후)을 계산합니다.

### 행동 유형 판정 우선순위

1. **Starter / Reset** 우선 적용
   - 3M 평균 < 3.0 **OR** 습관 시스템 < 3.0 **OR** 전체 응답 평균 < 3.0
2. **Developer**
   - Mapping, Making, Meshing이 **모두 3.7 이상** **AND** 세 점수의 차이가 **0.7 이하**
3. 그 외에는 3축 중 **가장 약한 축** 기준으로 결정
   - Mapping이 가장 약함 → **Drifter**
   - Making이 가장 약함 → **Dreamer**
   - Meshing이 가장 약함 → **Driver**

### 작심삼일 위험도

```
risk = (5 - making) * 0.25
     + fear        * 0.20
     + comfort     * 0.20
     + (5 - habit) * 0.25
     + topObstacle * 0.10
```

- 1.0 ~ 2.0 → **낮음**
- 2.1 ~ 3.0 → **보통**
- 3.1 ~ 4.0 → **높음**
- 4.1 ~ 5.0 → **매우 높음**

각 구간마다 사용자를 비난하지 않는 톤의 해석 문구가 함께 제공됩니다.

### 주요 장애물

7개 장애물 점수 중 가장 높은 항목을 "현재 주요 장애물"로 표시합니다.

## 10. 개인정보 보호 원칙

이 진단은 **이름, 연락처, 이메일, 학교, 직장, 정확한 나이, 수입, 정치 성향,
종교, 건강 정보, 가족 정보** 등 개인을 특정할 수 있는 정보를 어떠한 경로로도
요구하지 않습니다.

- 기본 동작에서 모든 답변과 실행계획은 **사용자의 브라우저(localStorage)에만**
  저장됩니다.
- 서버로 답변이 전송되지 않습니다.
- 비밀번호 입력 기록은 저장하지 않습니다.
- Supabase 저장 옵션을 켜더라도(아래 11번 참고) 개인식별정보는 절대 저장하지
  않으며, 사용자가 명시적으로 동의한 경우에만 익명 ID 기준으로 저장합니다.

진단을 중단하거나 모든 데이터를 지우려면 결과/완료 화면의 **"다시 시작"**
버튼을 누르거나, 브라우저 설정에서 사이트 데이터를 삭제하면 됩니다.

## 11. Supabase 연결 방법 (선택)

Supabase 연동은 **완전히 옵션**입니다. 환경변수가 없으면 자동으로 비활성화되며,
앱은 localStorage만으로 정상 작동합니다.

연결하려면 `.env.local`에 다음을 추가합니다.

```bash
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
```

그리고 `lib/supabase.ts`의 주석 처리된 `createClient` 호출을 활성화한 뒤
원하는 테이블 스키마(예: `responses`)를 직접 정의해 추가 구현하면 됩니다.

저장 가능한 데이터(권장):

- `anonymous_id` (UUID, 클라이언트에서 생성)
- `created_at`
- `scores` (영역 점수 JSON)
- `result_type`
- `risk_level`
- `top_obstacle`
- `action_plan` (선택 항목만, 자유 텍스트는 신중히)

**절대 저장 금지:**
- 이름, 이메일, 전화번호, IP 직접 수집, 학교, 직장, 종교, 정치 성향

저장 동의는 반드시 **기본값 OFF**의 명시적 체크박스로 구현하세요.
